# 13 — PDF Generation

## Use Cases

| Document | Trigger | Generated by |
|----------|---------|--------------|
| **Clearance Certificate** | All 4 departments cleared | `PdfService::generateClearance($clearance)` |
| **Document Release Slip** (optional) | Stage = released | `PdfService::generateReleaseSlip($request)` |
| **Payment Receipt Confirmation** (optional) | Payment approved | `PdfService::generatePaymentConfirmation($payment)` |

For v1, focus on the Clearance Certificate. The others are optional.

## Library

**`barryvdh/laravel-dompdf`** — wraps DomPDF, integrates with Laravel views.

```bash
composer require barryvdh/laravel-dompdf
```

Why DomPDF over Browsershot/Puppeteer:
- No Chromium installation required on the VPS.
- Sufficient for simple A4 layouts with text + images.
- Smaller Docker image.

## PDF Service

```php
// app/Services/PdfService.php
namespace App\Services;

use App\Models\Clearance;
use Barryvdh\DomPDF\Facade\Pdf;
use Illuminate\Support\Facades\Storage;

class PdfService
{
    public function generateClearance(Clearance $clearance): string
    {
        $clearance->load([
            'user',
            'documentRequest.documentType',
            'teacherSigner',
            'deanSigner',
            'accountingSigner',
            'saoSigner',
        ]);

        $pdf = Pdf::loadView('pdf.clearance', [
            'clearance' => $clearance,
            'student' => $clearance->user,
            'generatedAt' => now(),
        ])
        ->setPaper('a4', 'portrait')
        ->setOption('isRemoteEnabled', true);

        $path = "pdfs/clearance/{$clearance->id}.pdf";
        Storage::disk('local')->put($path, $pdf->output());

        $clearance->update(['pdf_path' => $path]);

        return $path;
    }
}
```

## Generation Triggers

The PDF is generated **automatically** when the clearance overall_status transitions to `completed`:

```php
// In ClearanceService
if ($clearance->fresh()->isComplete()) {
    $clearance->update(['overall_status' => 'completed', 'completed_at' => now()]);
    app(PdfService::class)->generateClearance($clearance);
    $clearance->user->notify(new ClearanceCompletedNotification($clearance));
}
```

## Download Route

```php
// routes/web.php
Route::middleware('auth')->get('/files/clearance/{clearance}/pdf', [FileController::class, 'clearancePdf'])
    ->name('files.clearance-pdf');

// Controller
public function clearancePdf(Clearance $clearance)
{
    $this->authorize('downloadPdf', $clearance);

    abort_unless($clearance->pdf_path, 404, 'PDF not yet generated');

    return Storage::disk('local')->download(
        $clearance->pdf_path,
        "clearance-{$clearance->user->student_id}-{$clearance->id}.pdf"
    );
}
```

## Verification (Stretch)

A public route `/verify/{clearance}` shows a minimal page confirming the clearance is genuine. Useful if students forward the PDF to other institutions.

```php
Route::get('/verify/{clearance}', function (Clearance $clearance) {
    return Inertia::render('Public/VerifyClearance', [
        'clearance' => [
            'student_name' => $clearance->user->fullname,
            'completed_at' => $clearance->completed_at?->toFormattedDateString(),
            'is_authentic' => $clearance->overall_status === 'completed',
        ],
    ]);
});
```

## Caveats

- DomPDF doesn't support modern CSS (no flex, no grid). Use tables for layout.
- Always inline styles or use `<style>` in the head — external stylesheets unreliable.
- Use absolute paths (`public_path()`, `storage_path()`) for images.
- Test PDF rendering in CI by generating one and snapshot-comparing the file size.
